# UX Design Specification: Outvestments (Plan B)

**Author:** Matt
**Date:** 2025-12-27
**Variant:** Alternate Approach for Comparison

---

## 1. Design Philosophy (Plan B)

### Core Principles

| Principle | Description |
|-----------|-------------|
| **Arcade cabinet, not sports bar** | 80s/90s arcade aesthetic - neon, pixel art accents, CRT glow |
| **Command-driven power user** | Keyboard-first with Cmd+K palette, minimal mouse dependency |
| **Single-screen density** | Everything visible at once, no page transitions for core flow |
| **Racing game HUD** | Position-based rankings, speedometer-style gauges, lap times |
| **Mobile as companion app** | Desktop is primary; mobile is monitoring/alerts only |

### Visual Inspiration

- Arcade cabinets (neon glow, scanlines, pixel fonts)
- Racing game HUDs (position indicators, lap splits, speedometers)
- Bloomberg Terminal (dense information, keyboard shortcuts)
- Cyberpunk 2077 UI (angular, neon accents on dark)
- Figma/Linear (command palette, minimal chrome)

### What to Avoid

- Casual mobile game look
- Too much whitespace (wastes screen real estate)
- Mouse-heavy workflows
- Loading states between core actions

## 2. Information Architecture (Plan B)

### Shot-First Hierarchy

**Key Difference:** Plan A is Target â†’ Aim â†’ Shot. Plan B inverts to Shot-first, because shots are the action.

| Level | Term | Description |
|-------|------|-------------|
| **1** | **Shot** | The position you're holding RIGHT NOW |
| **2** | **Aim** | The price prediction attached to that shot |
| **3** | **Target** | The thesis that groups related aims |

### Why Shot-First?

- Users care most about "what am I holding?"
- Active positions = active monitoring
- Thesis is context, not the main event
- Faster to "fire first, organize later"

### Quick-Fire Mode

Allow users to take a shot WITHOUT setting up a full thesis first. Assign to thesis later (or auto-create one).

```
QUICK SHOT:
[NVDA] [$1000] [BUY] [MARKET] â†’ FIRE

"We'll ask about your thesis later. Take the shot."
```
